# CLI

<Callout>Pumki UI uses the ShadCN CLI under the hood to manage components and setup.</Callout>

You can use the CLI to:

- Initialize your project
- Add components on demand
- Customize theme setup and structure

---

## Initialize Your Project

Use the `init` command to bootstrap your project with required configuration, utilities, and dependencies.

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']} defaultIndex="1">
  <Tabs.Tab>
    <CodeBlock language="bash">
      {`pnpm dlx shadcn@latest init`}
    </CodeBlock>
  </Tabs.Tab>
  <Tabs.Tab>
    <CodeBlock language="bash">
      {`npx shadcn@latest init`}
    </CodeBlock>
  </Tabs.Tab>
  <Tabs.Tab>
    <CodeBlock language="bash">
      {`yarn dlx shadcn@latest init`}
    </CodeBlock>
  </Tabs.Tab>
  <Tabs.Tab>
    <CodeBlock language="bash">
      {`bunx shadcn@latest init`}
    </CodeBlock>
  </Tabs.Tab>
</Tabs>

<br/>
{<h2>Options</h2>}

<br />

<CodeBlock language="bash">
{`Usage: shadcn init [options] [components...]

Options:
  -y, --yes              Skip confirmation (default: true)
  -d, --defaults         Use default config
  -f, --force            Overwrite existing setup
  -c, --cwd <cwd>        Set working directory
  -s, --silent           Suppress output
  --src-dir              Use src/ directory
  --no-src-dir           Avoid src/ directory
  --css-variables        Enable CSS theming (default: true)
  --no-css-variables     Disable CSS variables
  -h, --help             Show help`}
</CodeBlock>

## Add Components

Use the `add` command to include any component from the registry into your project.

<Tabs items={['Example: Add Button', 'Example: Add Accordion']} defaultIndex="0">
  <Tabs.Tab>
    <CodeBlock language="bash">
      {`npx shadcn@latest add button`}
    </CodeBlock>
  </Tabs.Tab>
  <Tabs.Tab>
    <CodeBlock language="bash">
      {`npx shadcn@latest add accordion`}
    </CodeBlock>
  </Tabs.Tab>
</Tabs>

<br/>
{<h2>Options</h2>}

<br />

<CodeBlock language="bash">
{`Usage: shadcn add [options] [components...]

Options:
  -y, --yes              Skip confirmation
  -o, --overwrite        Overwrite existing files
  -a, --all              Add all components
  -p, --path <path>      Custom output path
  -c, --cwd <cwd>        Working directory
  -s, --silent           Suppress output
  --src-dir              Use src/ directory
  --no-src-dir           Avoid src/ directory
  --css-variables        Enable CSS variables
  --no-css-variables     Disable CSS variables
  -h, --help             Show help`}
</CodeBlock>

## Build Registry (Optional)

If you're self-hosting or modifying your registry, use the `build` command to generate JSON from your local `registry.json`.

<Tabs items={['Basic Build', 'Custom Output']} defaultIndex="0">
  <Tabs.Tab>
    <CodeBlock language="bash">
      {`npx shadcn@latest build`}
    </CodeBlock>
  </Tabs.Tab>
  <Tabs.Tab>
    <CodeBlock language="bash">
      {`npx shadcn@latest build --output ./public/registry`}
    </CodeBlock>
  </Tabs.Tab>
</Tabs>

<br/>
{<h2>Options</h2>}

<br />

<CodeBlock language="bash">
{`Usage: shadcn build [options] [registry]

Arguments:
  registry               Path to registry.json (default: ./registry.json)

Options:
  -o, --output <path>    Output directory (default: ./public/r)
  -c, --cwd <cwd>        Working directory
  -h, --help             Show help`}
</CodeBlock>
